### Optimal Parameters Found:
- baseline_lam: 4.16e5 (narrower than initial 1e6)

- Ridge alpha: ~6854 (similar range to ML-002)

### Learnings:
- Two-stage optimization is recommended over joint optimization for large search spaces
- Stage 1 uses 30% subsampling to speed up preprocessing optimization (the bottleneck)
- Preprocessing optimization provides modest gains (~0.8%) when starting from a good baseline
- Narrowing parameter search ranges (1e5-1e7 for baseline_lam) improves convergence
- savgol_window_half trick ensures odd window sizes: window = 2*half + 1
- Higher savgol_polyorder (4 vs 3) captures more spectral detail
- Higher baseline_p (0.026 vs 0.01) means less aggressive baseline correction

## ML-004 - Add feature selection methods to optimization
**Status**: PASSED
**Date**: 2026-01-28
**RMSE_mean**: 2.907 (no regression from baseline)

### Changes Made:
- Added wavelength selection as alternative feature extraction method in src/features.py:
  - `select_wavelengths()` supports correlation, variance, f_score methods
  - `_correlation_column_wise()` for memory-efficient correlation computation
- Added `optimize_preprocessing_and_features()` in src/optimization.py for Stage 1 with feature method search
- Added `optimize_model_with_features()` in src/optimization.py for Stage 2 with feature method
- New CLI flags:
  - --feature_method: Choose between pca or wavelength_selection
  - --selection_method: Wavelength selection method (correlation, variance, f_score)
  - --n_selected_wavelengths: Number of wavelengths to select (default 500)
  - --n_selected_min, --n_selected_max: Search range for optimization
  - --optimize_features: Enable feature method search in two-stage optimization

### Optimal Parameters Found:
- Feature method: PCA (outperforms wavelength_selection on this dataset)
- n_components: 86-99 (consistent with ML-003)
- Ridge alpha: ~5700-6854 (consistent with ML-003)
- Preprocessing params: Similar to ML-003 baseline

### Learnings:
- PCA consistently outperforms direct wavelength selection for this spectral dataset
- Wavelength selection trials (correlation, variance, f_score) showed RMSE 2.5-3.0+ on subsampled data
- PCA trials showed RMSE ~2.06-2.12 on subsampled data
- Memory-efficient correlation computation avoids creating (n_features+1)x(n_features+1) matrix
- Adding --optimize_features increases search space significantly; recommend more trials (30+)
- Feature method choice is dataset-dependent; wavelength selection may work better on other datasets
- Direct wavelength selection requires higher regularization (ill-conditioned matrix warnings)

## ML-005 - Add ensemble model support
**Status**: PASSED
**Date**: 2026-01-28
**RMSE_mean**: 2.907 (no regression from baseline)
**Ensemble CV Score**: ~2.86

### Changes Made:
- Added `get_ensemble_model()` in src/models/traditional.py for creating ensemble with default params
- Added `get_optimized_ensemble_model()` for optimizing base models before combining
- New CLI flags:
  - --ensemble: Enable ensemble mode
  - --ensemble_method: Choose between stacking (meta-learner) or voting (average)
  - --ensemble_models: Specify base models (default: pls ridge rf)

### Ensemble Architecture:
- Stacking: Uses StackingRegressor with Ridge as final estimator
- Voting: Uses VotingRegressor to average predictions
- For multi-target: Wrapped with MultiOutputRegressor
- Base models optimized individually via Optuna before combining

### Optimal Parameters Found:
- PLS n_components: 12-19
- Ridge alpha: ~6854
- RF: n_estimators=72, max_depth=14, min_samples_split=2, min_samples_leaf=4, max_features=None

### Learnings:
- Stacking ensemble with optimized base models achieves CV score ~2.86
- Final evaluation RMSE (~3.0) slightly higher than CV score due to overfitting risk
- Ensemble doesn't always outperform optimized single models on small datasets
- Base model optimization before ensemble is critical for good performance
- MultiOutputRegressor wrapper needed for multi-target regression
- Ensemble benefits: robustness across different targets, reduced variance
- For this dataset, optimized Ridge alone performs comparably to ensemble
